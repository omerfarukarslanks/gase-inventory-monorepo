# Done Checklist

- Confirm the user job, actor, and permission rule.
- Confirm the chosen file placement matches the current monorepo boundaries.
- Cover loading, empty, error, disabled, and success states.
- Reuse an existing domain pattern before inventing a new component shape.
- Keep terminology aligned with the matching flow on the other surface.
- Keep tokens in `packages/design-tokens` and translations in `packages/i18n` when shared.
- Avoid adding fresh duplication near `packages/core` or `apps/web/lib`.
- Run lint and typecheck for the touched app when possible.
- Manually exercise the main user flow when possible.
