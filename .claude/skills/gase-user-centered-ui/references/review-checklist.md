# Review Checklist

## Task Clarity

- Is the main user task obvious within a few seconds?
- Is there a clear primary action?
- Does the screen explain what changed after an action completes?

## Information Hierarchy

- Is the most important information visible without extra scanning?
- Are secondary details visually quieter?
- Are related fields and actions grouped together?

## Forms and Inputs

- Does each field earn its place?
- Are defaults, placeholders, helper text, and validation messages specific?
- Can the user recover from errors without losing work?

## Data Views

- Can a user scan rows, statuses, totals, and actions quickly?
- Are filters easy to discover and reset?
- Is the empty state informative instead of blank?

## Feedback and Trust

- Is loading visible?
- Are failures actionable?
- Are destructive actions separated and explicit?
- Does the interface avoid hidden state changes?

## Accessibility

- Are controls labeled?
- Is status communicated by text, not only color?
- Is the interaction usable with keyboard on web and with touch on mobile?

## Cross-Surface Consistency

- Do web and mobile use the same domain language?
- Do both surfaces expose the same critical outcome and safeguards?
